OpenSC: CKR_FUNCTION_NOT_SUPPORTED : as of version 0.19.0

How to interpret the following: Usually, functionality/ functions like e.g. C_Login is used by calls into opensc-plcs11.so/.dll etc.
If OpenSC doesn't implement a function, then there usually is no imlementation in the driver either, but there are exceptions: E.g. C_Encrypt is not omplemented, but the driver offers Encrypt/Decrypt via
libopensc's sc_card_ctl function. Look at src/constants_types.rs and import that: SC_CARDCTL_ACOS5* are the command encodings for functionality accessible from the driver directly (it's extensively used by acos5_64_gui)
